## 3. 已知坑/特殊处理

### 3.1 显式 signingConfig 必须配双路径

`app/build.gradle.kts` 里 signingConfigs.create("stingDebug") 用 `storeFile = file("debug.keystore")`(相对路径 → `app/debug.keystore`)。
**所以 build.yml 必须同时把 keystore 解到 `app/` 和 `~/.android/`**,否则 AGP 报"Keystore file not found"。

教训:以后所有 v2.0+ 项目都**直接用 AGP 默认(走 `~/.android/debug.keystore`)**,不再显式配 signingConfigs。

### 3.2 NAS SMB 同步慢/卡

跟 OverTimeWageCalc 一样,大项目源码同步 NAS 经常 timeout(124)。**改用方案**:
- 重要源文件单独 sync(只 24 个文件)
- 如果再卡,**接受 GitHub 仓库作为单一真实源**,sting 必要时 `git clone`

### 3.3 git_api_push.py 拿不到新建空仓库的 ref

新建空仓库(API `auto_init=false` 创建),`GET /git/ref/heads/main` 返 409 "Git Repository is empty"。
**绕开**:用 `git push -u origin main` 加 x-access-token URL 直推。
**修法**:把 git_api_push.py 加 fallback:`git ref = None` 时 `subprocess.run('git push -u origin main')`

## 4. 错误日志

### 4.1 Actions run 27596563060(首次编译失败)

错误:`Keystore file '/home/runner/work/WageDiary/WageDiary/app/debug.keystore' not found for signing config 'stingDebug'.`
根因:build.yml 只把 keystore 解到 `~/.android/debug.keystore`,但 build.gradle.kts 配的 `signingConfigs.stingDebug` 走 `file("debug.keystore")` (即 `app/debug.keystore`)。
修复:build.yml 加 `mkdir -p app && base64 -d .github/keystore.b64 > app/debug.keystore`
结果:run 27596899727 编译成功。

### 4.3 v1.0 三个严重 bug(sting 实测,2026-06-16 反馈)

**Bug A:额外加班/备注点输入框弹不出历史**
- 原因:OutlinedTextField 内部的 click handler 消费点击事件,外层 Modifier.clickable 永远收不到
- 解决:新增 HistoryInputRow 组件(只读显示当前值 + 显式"选历史"按钮 → ModalBottomSheet)
- 额外加班/备注各显示一个"选历史"按钮 + 下方仍可手动输入

**Bug B:导出 Excel 崩溃 `java.lang.NoClassDefFoundError: java.awt.font.FontRenderContext`**
- 原因:`autoSizeColumn` → `SheetUtil.copyRowStyle` → `FontRenderContext`,Android 无 awt
- 解决:删 `autoSizeColumn`,改手动列宽 `setManualColumnWidths`
- **POI Android 铁律**:**禁用** 任何依赖 java.awt.Font* / Graphics* 的 API(SheetUtil.copyRowStyle 等)

### 4.4 FTP 优先,5 分钟超时切 SMB(2026-06-16)

`scripts/nas_client.py` 统一封装,FTP 通则 FTP(快),FTP 5 分钟 timeout 切 SMB 重试 1 次,SMB 也失败抛 NASError 不重试。
